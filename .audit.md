# Arenibus Marketing Audit — SEO, Performance, Accessibility & Quality

## Project Overview

**Arenibus Marketing Website** is a static Next.js 16 marketing site for the Arenibus nephrology information system (MVP phase).

| Layer | Technology | Location |
|-------|------------|----------|
| Framework | Next.js 16 (App Router, static export) | `app/` |
| Language | TypeScript | `*.ts`, `*.tsx` |
| Styling | Tailwind CSS v4 | `app/globals.css` |
| Assets | Static images, SVG favicon | `public/` |
| Build output | Static HTML/JS/CSS | `out/` |
| Hosting | WebSupport SSH (static files) | `deploy.sh`, `deploy.bat` |
| Primary URL | https://arenibus.polascin.net/ | — |

Key pages:

- `app/page.tsx` — single-page marketing landing page (hero, features, demo CTA, contact)
- `app/layout.tsx` — root layout with metadata and theme provider
- `app/globals.css` — design tokens, light/dark theme CSS variables
- `next.config.ts` — static export configuration
- `public/favicon.svg` — branded SVG favicon
- `public/apple-touch-icon.png` — Apple touch icon

## 🎯 Agent Goals

Autonomous agent for deep, recurring audits of the Arenibus marketing site. Covers:

1. **SEO discoverability** — metadata, Open Graph, structured data, sitemap, robots, canonical URLs, heading hierarchy, internal links.
2. **Performance** — Core Web Vitals, bundle size, image/font optimization, static export efficiency.
3. **Accessibility (WCAG 2.1 AA)** — color contrast, keyboard navigation, focus indicators, screen-reader semantics, motion preferences.
4. **Code quality** — TypeScript strictness, Tailwind patterns, Next.js idioms, dead code, dependency hygiene.
5. **Security of static export** — hardcoded secrets, unsafe inline scripts, CSP considerations, dependency vulnerabilities.
6. **Content correctness** — Slovak copy accuracy, brand consistency, demo integration, contact details.
7. **Deployment health** — build succeeds, output is valid, deploy script works, live site matches build.

**Every run ends with a self-improvement step**: record new patterns, false positives, and newly discovered project-specific checks back into this file so the next audit is sharper.

## 🔍 Primary Tasks

### 1. SEO Audit

SEO is the primary focus of this audit. The marketing site must rank and share correctly.

#### 1.1 Metadata (`app/layout.tsx`, `app/page.tsx`)

- **Title tag:** unique, descriptive, ≤ 60 chars, includes "Arenibus".
  - Current expected: `Arenibus – Informačný systém pre nefrologické ambulancie`
- **Meta description:** unique per page, ≤ 160 chars, actionable.
- **Canonical URL:** present and points to `https://arenibus.polascin.net/`.
- **Language / locale:** `lang="sk"` or `lang="sk-SK"` set on `<html>`.
- **Open Graph:** `og:title`, `og:description`, `og:type`, `og:url`, `og:image`.
  - `og:image` must be an absolute URL (`https://arenibus.polascin.net/...`) with recommended 1200×630 px.
- **Twitter Cards:** `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`.
- **Robots meta:** no accidental `noindex` in production.
- **Viewport & theme-color:** present.

#### 1.2 Favicon & PWA basics

- `favicon.ico` or `favicon.svg` linked correctly.
- `apple-touch-icon.png` linked via `<link rel="apple-touch-icon" />`.
- Recommended: `manifest.json` with theme/background colors and icons.
- Theme color matches brand (`#0f766e`).

#### 1.3 Heading hierarchy (`app/page.tsx`)

- Exactly one `<h1>` per page.
- Heading levels must not skip (`h1 → h2 → h3`, never `h1 → h3`).
- Feature section headings use semantic heading tags, not styled `<div>`.
- No empty headings.

#### 1.4 Structured data

- Add `application/ld+json` JSON-LD for `SoftwareApplication` or `Organization`.
  - Required fields: `@context`, `@type`, `name`, `description`, `url`, `applicationCategory`, `offers`/`isAccessibleForFree`.
  - Optional but recommended: `logo`, `sameAs`, `provider`.

#### 1.5 Sitemap & robots

- `public/robots.txt` exists and allows all public paths:
  ```
  User-agent: *
  Allow: /
  Sitemap: https://arenibus.polascin.net/sitemap.xml
  ```
- `public/sitemap.xml` exists and lists the canonical homepage URL with `<lastmod>` and `<priority>`.
- **Note:** Next.js static export does not auto-generate sitemap/robots; they must live in `public/`.

#### 1.6 URL & link hygiene

- All internal anchors use descriptive text (no "klikni tu" / "viac").
- External links (`demo.arenibus.com`) open in new tab with `rel="noopener noreferrer"`.
- No broken fragment identifiers (`#features`, `#demo`, `#contact` must exist).
- Contact e-mail uses `mailto:arenibus@polascin.net`.

#### 1.7 Image SEO

- Logo and icons have meaningful `alt` text.
- Decorative images use empty `alt=""`.
- Images are not rendered as text-only `<svg>` without accessible labels.

### 2. Performance Audit

- **Core Web Vitals targets:**
  - LCP ≤ 2.5 s
  - INP ≤ 200 ms
  - CLS ≤ 0.1
  - TTFB ≤ 600 ms
- **Bundle:** run `npm run build` and inspect `out/_next/static/...`.
  - No oversized JS chunks.
  - No duplicate icon libraries.
- **Images:**
  - Use Next.js `<Image>` where possible (note: `next/image` with static export may require `unoptimized: true` or loader config).
  - Provide `width`/`height` to prevent CLS.
  - Use WebP/AVIF where appropriate.
- **Fonts:**
  - Self-host or preconnect to font CDN.
  - Use `font-display: swap`.
- **CSS:**
  - Tailwind v4 `@import "tailwindcss"` is correct.
  - No unused custom CSS bloating the bundle.
- **Third-party scripts:** only essential scripts; load async/defer.

### 3. Accessibility (WCAG 2.1 AA)

- **Contrast:**
  - Text on brand teal (`--brand: #0f766e`) must meet 4.5:1 for normal text.
  - `--brand-text` (white in light, dark in dark) on brand backgrounds must meet 4.5:1.
  - Links must be distinguishable from body text.
- **Focus:**
  - All interactive elements have visible focus rings.
  - Focus order follows DOM order.
- **Keyboard:**
  - All buttons and links reachable via Tab.
  - Mobile menu toggle works with Enter/Space.
  - Escape closes mobile menu.
- **Screen readers:**
  - Dark mode toggle has `aria-label`.
  - Mobile menu button has `aria-expanded` and `aria-controls`.
  - SVG icons have `aria-hidden` or meaningful labels.
- **Motion:**
  - Respect `prefers-reduced-motion` for any transitions/animations.
- **Language:**
  - `<html lang="sk">` so screen readers use Slovak pronunciation.

### 4. Code Quality Audit

#### 4.1 TypeScript

- `strict: true` enabled in `tsconfig.json`.
- No implicit `any`.
- Components have explicit return types or inferred ReactElement.
- No `// @ts-ignore` or `// @ts-nocheck`.

#### 4.2 Next.js / React

- App Router conventions: default exports for pages, correct metadata API.
- No class components.
- Hooks rules respected (no conditional hooks).
- Client components (`"use client"`) only where necessary (theme toggle, localStorage).
- Server Components used by default.

#### 4.3 Tailwind CSS

- Use arbitrary values sparingly; prefer design tokens from `globals.css`.
- No conflicting Tailwind classes on the same element.
- Dark mode uses `data-theme="dark"` and CSS variables, not `dark:` class variants.
- Verify `--color-*` theme registration matches variables used in JSX.

#### 4.4 Dead / unused code

- No commented-out components or imports.
- No unused public assets.
- No dependencies in `package.json` that are not imported.

#### 4.5 Lint

- Run `npm run lint` and fix all errors/warnings.

### 5. Security Audit (Static Export)

- **Secrets:** grep for `password`, `secret`, `api_key`, `token`, `private` in code.
  - Must not find hardcoded credentials.
- **Client-side exposure:** no sensitive logic in client components.
- **CSP readiness:** static export cannot set headers; document any inline scripts and recommend moving them to external files.
- **Dependencies:**
  - `npm audit` should pass or have documented acceptable exceptions.
  - Keep Next.js and React up to date.
- **Links:** external links use `rel="noopener noreferrer"`.
- **Forms:** contact form does not submit to insecure endpoint; if backendless, it must clearly indicate that it is informational or use a safe form service.

### 6. Content & Brand Consistency Audit

- **Brand alignment:** colors, logo, and tone match `https://demo.arenibus.com/`.
- **Demo section:**
  - Link to `https://demo.arenibus.com/` is correct and reachable.
  - Demo credentials `demo-lekar` / `demo-sestra` are accurate.
- **Contact:** e-mail `arenibus@polascin.net` is consistent everywhere.
- **Slovak copy:**
  - Correct diacritics and grammar.
  - Consistent terminology (e.g., "nefrologická ambulancia", "informačný systém").
- **Feature list:** reflects actual MVP features documented in the core Arenibus system.
- **Copyright / footer:** matches demo site format.

Severity definitions:

- **Critical:** broken deploy, build failure, hardcoded secret, security vulnerability, site unavailable.
- **High:** missing canonical/OG/sitemap, severe accessibility blocker, major performance regression.
- **Medium:** heading hierarchy issue, contrast near threshold, missing alt text, dead code.
- **Low:** minor copy inconsistency, missing manifest, suboptimal image format.
- **Info:** suggestions and future improvements.
